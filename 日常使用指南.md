# 📅 Cloudflare Tunnel 日常使用指南

## 🌅 每日启动流程（3分钟）

### 早上开机后：

1. **双击运行** `一键启动.bat`

2. **等待启动完成**（约10秒）
   - 开发服务器启动
   - Cloudflare Tunnel 连接

3. **复制新地址**
   ```
   https://xxx-xxx-xxx.trycloudflare.com
   ```

4. **分享给需要的人**
   - 发送链接到微信群
   - 发送给客户/同事
   - 任何人都能访问

5. **开始工作**
   - 修改代码
   - 效果实时同步
   - 无需额外操作

---

## 🌙 每晚关闭流程（10秒）

1. **关闭 Cloudflare Tunnel 窗口**
   - 在显示公网地址的窗口中
   - 按 Ctrl+C

2. **关闭开发服务器窗口**
   - 在 `npm run dev` 窗口中
   - 按 Ctrl+C

3. **正常关机**（可选）

---

## 💡 重要提示

### ⚠️ 每天都会生成新地址

**问题**: 每次重启得到不同的随机域名

**解决方案**:
1. **提前告知**: "地址每天会变化"
2. **固定时间**: 每天早上重新发送新地址
3. **专业方案**: 考虑使用 Vercel 获得固定域名

### ✅ 代码修改实时同步

**工作流程**:
```
修改代码 → 保存 (Ctrl+S) → 自动编译 → 自动刷新 → 实时生效
```

**时间要求**:
- 小修改: 1-3 秒
- 大修改: 5-10 秒
- **完全自动化**: 无需手动操作

## 🔧 故障排除

### 问题1: 链接无法访问
**解决**:
1. 检查两个窗口是否都在运行
2. 检查是否有网络连接
3. 重新启动系统

### 问题2: 修改后看不到效果
**解决**:
1. 等待 3-5 秒让编译完成
2. 在浏览器中按 Ctrl+R 刷新
3. 清除浏览器缓存

### 问题3: 隧道经常断开
**解决**:
1. 检查网络连接稳定性
2. 考虑使用 Vercel 部署
3. 或使用 ngrok 作为备选
