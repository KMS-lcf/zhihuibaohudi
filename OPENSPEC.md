# 🚀 OpenSpec Quick Start Guide

OpenSpec is now initialized in this project! This guide will help you get started with the OpenSpec workflow.

## ✨ What is OpenSpec?

OpenSpec is a structured workflow for managing changes, proposals, and implementations in your project. It provides:

- **🤔 Explore Mode** - Think through ideas and investigate problems
- **📝 Propose Mode** - Create formal proposals with design and specs
- **⚡ Apply Mode** - Implement changes step by step
- **✅ Archive Mode** - Complete and archive changes

## 🎯 Quick Start

### 1. Start a New Feature or Change

**Step 1: Explore**
```
Use the explore mode to brainstorm and clarify requirements
```

**Step 2: Propose**
```
Create a formal proposal with design, specs, and implementation tasks
```

**Step 3: Apply**
```
Implement the proposal using the structured workflow
```

**Step 4: Archive**
```
Complete the change and archive it for reference
```

## 📁 Where Files Live

```
.openspec/
├── changes/     → Active changes you're working on
├── proposals/   → Draft and proposed changes
├── archived/    → Completed changes
└── templates/   → Templates for proposals and changes
```

## 💡 Tips

- Use **explore mode** when you're unsure about the approach
- Use **propose mode** when you have a clear idea and want to formalize it
- Use **apply mode** when you're ready to implement
- Use **archive mode** when a change is complete and deployed

## 🛠️ Configuration

The OpenSpec configuration is in `.openspec/config.json`. You can customize:
- Workflow settings
- Template locations
- Directory paths
- Project metadata

## 📚 Templates

Templates are available in `.openspec/templates/`:
- `proposal.md` - For creating new proposals
- `change.md` - For tracking implementation progress

Feel free to customize these templates to match your project's needs!
